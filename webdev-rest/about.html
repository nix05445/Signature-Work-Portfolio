About the Project

 

 

APIs and Frameworks Used

 

 

Leaflet

An open-source JavaScript library used for
mobile-friendly interactive maps. This API
works across all device types and it's
source code makes it easy to work with!


 

Nominatim

Used to search open street map data
by name and address or geocoding. It can
also find a location on the map based on
the latitude and longitude coordinates!
Some key supporters include OpenCage,
Grasshopper, and Komoot.


 

 

Our own Rest API

Cooked up on our very own machines, our
rest_server.mjs API is seriously (or at least we
think so) one of a kind. It allows
us to easily talk back and forth with our
crime database, we can use it to delete,
insert, and fetch data.


 

 

Vue.js

An open-source modelview viewmodel front end
JavaScript library for building user interfaces
and single-page applications.


 

 

 

 

Foundation CSS

A free responsive front-end framework, providing
a responsive grid and HTML and CSS UI
components, templates, and code snippets,
including typography, forms, buttons and other
interface elements, as well as optional
functionality provided by JavaScript extensions.


 

 

    Six Interesting Things About Using the Application

     

    1. Thanks to Nominatim, this application could work with any major city, as long as we have an appropriate crime datadase for it
    2. Leaflet has a ton of cool functions/features you can use, but the list is so vast I'd be amazed to see what leaflet CAN'T do.
    3. Vue.js has a ton of sweet in-app features that made grabbing stuff from API's super easy.
    4. Most of the crime in St. Paul occurs right near the capital.
    5. The latitude/longitude coordinates do not change a ton as you traverse around the map, meaning you have to be ultra precise in how you search.
    6. The northeast portion of St. Paul tends to be safest, as the number of crimes drops in those neighborhoods compared to others.
